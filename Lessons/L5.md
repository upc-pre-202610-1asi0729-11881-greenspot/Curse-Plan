## Lección 5 — Fundamentos de Programación Orientada a Objetos (POO)

Hasta ahora nuestro código era una lista de instrucciones hacia abajo. Hoy vamos a subir de nivel y aprenderemos cómo hacen los profesionales para crear videojuegos reales usando **POO** (Programación Orientada a Objetos).

---

### ¿Qué es una clase y qué es un objeto?

En Java, no creamos personajes de la nada; primero necesitamos un "molde".

* **La Clase (El Molde):** Es un plano o plantilla. No tiene vida propia, pero define cómo será algo. En nuestro juego, la clase es `Personaje`.
* **El Objeto (La Creación):** Es cuando usamos ese molde para crear algo real en el juego. Usando el molde `Personaje`, podemos crear a un "Guerrero" o a un "Dragón Oscuro".

> **Ejemplo de la vida real:** La *clase* es la receta para hacer una torta de chocolate. El *objeto* es la torta real que te comes. Puedes usar una sola receta (clase) para hornear muchas tortas (objetos).

---

### Atributos y métodos dentro de una clase

Todo molde (clase) define dos cosas importantes para sus objetos:
1.  **Atributos:** Lo que el objeto *es* o *tiene* (Ej: vida, nombre, puntos de ataque).
2.  **Métodos:** Lo que el objeto *puede hacer* (Ej: atacar, curarse).

---

### Crear nuestra primera clase: el `Personaje`

Vamos a ver el corazón de nuestro simulador RPG. Aquí definimos nuestro molde. Fíjate que usamos la palabra `private` para proteger los datos (a esto le llamamos **Encapsulación**).

```java
abstract class Personaje {
    // 1. Atributos (privados para que nadie haga trampa con la vida)
    private String nombre;
    private int vida;
    private int ataque;

    // 2. Constructor (El preparador inicial)
    public Personaje(String nombre, int vida, int ataque, int pociones) {
        this.nombre = nombre;
        this.vida = vida;
        this.ataque = ataque;
    }

    // 3. Getters (Para poder leer los atributos privados)
    public String getNombre()  { return nombre; }
    public int getVida()       { return vida; }

    // 4. Métodos (Acciones)
    public abstract void atacar(Personaje enemigo);
}
