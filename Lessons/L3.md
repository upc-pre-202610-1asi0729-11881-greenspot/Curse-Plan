## Lección 3 — Estructuras de Control

### ¿Qué es una condición?

Imagina que estás en un videojuego y el juego te pregunta: "¿tienes vida suficiente para seguir?" Si la respuesta es sí, continúas. Si es no, game over. Eso es exactamente lo que hace un `if` en Java: evalúa una condición y decide qué camino tomar.

La estructura básica es:

```java
if (condición) {
    // se ejecuta si la condición es verdadera
} else {
    // se ejecuta si la condición es falsa
}
```

También puedes encadenar más de dos caminos con `else if`:

```java
if (vidaHeroe >= 80) {
    System.out.println("Salud alta — sin peligro");
} else if (vidaHeroe >= 40) {
    System.out.println("Salud media — ten cuidado");
} else {
    System.out.println("Salud crítica — ¡en peligro!");
}
```

Java evalúa las condiciones de arriba hacia abajo y entra en el primer bloque que sea verdadero. Los demás los ignora.

---

### Los operadores de comparación

Dentro del `if` usamos operadores para comparar valores:

| Operador | Significado | Ejemplo |
|---|---|---|
| `==` | ¿Son iguales? | `vida == 0` |
| `!=` | ¿Son distintos? | `vida != 100` |
| `>` | ¿Es mayor? | `ataque > 15` |
| `<` | ¿Es menor? | `vida < 30` |
| `>=` | ¿Es mayor o igual? | `turno >= 1` |
| `<=` | ¿Es menor o igual? | `vida <= 0` |

---

### Error frecuente: `=` vs `==`

Este es el error más común al empezar. Son dos cosas completamente distintas:

```java
vida = 100;   // ASIGNA el valor 100 a la variable vida
vida == 100;  // PREGUNTA si vida vale 100 (verdadero o falso)
```

Si escribes `if (vida = 100)` Java te dará un error de compilación. Siempre usa `==` dentro de un `if`.

---

### ¿Qué es un bucle while?

Un bucle `while` repite un bloque de código mientras una condición sea verdadera. Cuando la condición se vuelve falsa, el bucle se detiene y el programa continúa con lo que venga después.

```java
while (condición) {
    // se repite mientras la condición sea verdadera
}
```

En el juego, el combate continúa mientras ambos personajes tengan vida:

```java
while (vidaHeroe > 0 && vidaEnemigo > 0) {
    int danoHeroe = ataqueHeroe + rand.nextInt(6);
    vidaEnemigo   = vidaEnemigo - danoHeroe;
    System.out.println(nombreHeroe + " ataca por " + danoHeroe + " puntos!");

    if (vidaEnemigo > 0) {
        int danoEnemigo = ataqueEnemigo + rand.nextInt(8);
        vidaHeroe       = vidaHeroe - danoEnemigo;
        System.out.println(nombreEnemigo + " contraataca por " + danoEnemigo + " puntos!");
    }

    turno++;
}
```

---

### El operador && y otros operadores lógicos

| Operador | Significado | Ejemplo |
|---|---|---|
| `&&` | Y — ambas condiciones deben ser verdaderas | `vidaHeroe > 0 && vidaEnemigo > 0` |
| `\|\|` | O — basta con que una sea verdadera | `estaVivo \|\| tieneEscudo` |
| `!` | NO — invierte la condición | `!estaVivo` |

En el juego usamos `&&` porque el combate solo continúa si el héroe **y** el enemigo siguen vivos. Si uno cae, el bucle termina.

---

### El daño aleatorio con Random

En la Lección 2 el daño era fijo. En esta lección lo hacemos aleatorio usando `Random`:

```java
Random rand = new Random();
int dano = ataqueHeroe + rand.nextInt(6);
```

`rand.nextInt(6)` genera un número entre 0 y 5. Sumado al ataque base, el daño varía en cada turno, igual que al tirar un dado en un juego de rol.

---

### Errores comunes en esta lección

| Error | Causa | Cómo evitarlo |
|---|---|---|
| Usar `=` en vez de `==` en el `if` | Confundir asignación con comparación | Recuerda: `=` asigna, `==` compara |
| Bucle infinito | La condición nunca se vuelve falsa | Asegúrate de que la vida baje en cada turno |
| El enemigo ataca con 0 de vida | Falta el `if (vidaEnemigo > 0)` antes del contraataque | Encierra el contraataque en ese `if` |
| Olvidar `turno++` | El turno nunca avanza | Siempre incrementa el contador al final del bucle |

---

### Flujo completo del combate en esta lección

```
inicio
  └── while (vidaHeroe > 0 && vidaEnemigo > 0)
        ├── héroe ataca → vidaEnemigo baja
        ├── if (vidaEnemigo > 0)
        │     └── enemigo contraataca → vidaHeroe baja
        └── turno++
fin del while
  └── if (vidaHeroe > 0) → VICTORIA
      else               → DERROTA
```
