## Lección 4 — Métodos, Entrada/Salida e Introducción a la POO

### ¿Qué es un método y para qué sirve?

Un método es un bloque de código con nombre que realiza una tarea específica. La idea es simple: si vas a hacer lo mismo más de una vez, ponlo en un método.

Sin métodos, tendrías esto en cada turno:
```java
int dano = ataqueHeroe + rand.nextInt(6);
vidaEnemigo = Math.max(0, vidaEnemigo - dano);
System.out.println(nombreHeroe + " ataca a Dragón Oscuro por " + dano + " puntos!");
System.out.println("Dragón Oscuro le queda " + vidaEnemigo + " de vida.");
```

Con un método, en cada turno solo escribes:
```java
atacar(nombre, "Dragón Oscuro", ataqueHeroe, vidaEnemRef);
```

Mismo resultado, mucho más limpio.

### Anatomía de un método

```java
static void atacar(String atacante, String defensor, int ataque, int[] vidaDefensor) {
//     ↑       ↑         ↑                                                 ↑
//  tipo    nombre    parámetros (datos que recibe el método)          más parámetros
    int dano = ataque + rand.nextInt(6);
    vidaDefensor[0] = Math.max(0, vidaDefensor[0] - dano);
    System.out.println(atacante + " ataca a " + defensor + " por " + dano + " puntos!");
}
```

- `static` → el método pertenece a la clase, no a un objeto (lo veremos mejor en L5)
- `void` → el método no devuelve ningún valor
- Los parámetros son los datos que el método necesita para funcionar

### Métodos que devuelven un valor

Cuando un método necesita devolver un resultado, cambias `void` por el tipo de dato que devuelve y usas `return`:

```java
static int usarPocion(String nombre, int vida, int vidaMax, int[] pociones) {
    if (pociones[0] > 0) {
        int curacion = 30;
        vida = Math.min(vida + curacion, vidaMax);
        pociones[0]--;
        System.out.println(nombre + " usa una poción y recupera " + curacion + " de vida!");
    } else {
        System.out.println("¡No te quedan pociones!");
    }
    return vida;  // devuelve la vida actualizada
}
```

Lo llamas así y guardas el resultado:
```java
vidaHerRef[0] = usarPocion(nombre, vidaHerRef[0], vidaMaxHeroe, pocionesRef);
```

### Entrada del jugador con Scanner

`Scanner` es la herramienta estándar de Java para leer lo que el usuario escribe. Antes de usarlo hay que importarlo:

```java
import java.util.Scanner;
```

Luego lo creas y lo usas:

```java
Scanner scanner = new Scanner(System.in);

System.out.print("¿Cómo se llama tu personaje? ");
String nombre = scanner.nextLine();  // lee texto

System.out.print("Elige (1/2): ");
String clase = scanner.nextLine();   // lee la opción elegida
```

Al terminar el programa, es buena práctica cerrarlo:
```java
scanner.close();
```

### Comparar Strings: `.equals()` no `==`

Cuando el jugador escribe su elección, necesitas comparar el texto. En Java, los `String` **nunca** se comparan con `==`:

```java
// X Incorrecto — puede no funcionar
if (clase == "2") { ... }

// ✔ Correcto — siempre usa .equals()
if (clase.equals("2")) { ... }
```

Esto es porque `==` compara si dos variables apuntan al mismo lugar en memoria, no si tienen el mismo texto. `.equals()` compara el contenido.

### Primera mirada a la POO

En esta lección el héroe sigue siendo un conjunto de variables sueltas:

```java
String nombre   = "Aragorn";
int vidaHeroe   = 120;
int ataqueHeroe = 18;
int[] pociones  = {3};
```

Eso funciona, pero imagina que tuvieras 10 personajes. Serían 40 variables sueltas. En la Lección 5 agruparemos todo dentro de una clase:

```java
class Personaje {
    String nombre;
    int vida;
    int vidaMaxima;
    int ataque;
    int pociones;
}
```

Con eso, cada personaje es un solo objeto con todo adentro. El código hace exactamente lo mismo, pero es mucho más fácil de leer, modificar y escalar.

---

### Errores comunes en esta lección

| Error | Causa | Cómo evitarlo |
|---|---|---|
| Comparar Strings con `==` | Confundir comparación de objetos con comparación de texto | Siempre usa `.equals()` para Strings |
| Olvidar `scanner.close()` | Dejar recursos abiertos | Cierra el Scanner al final del programa |
| Método `void` con `return valor` | Inconsistencia entre tipo declarado y lo que devuelve | Si devuelve algo, cambia `void` por el tipo correcto |
| Llamar un método antes de declararlo | El compilador no lo encuentra | En Java el orden de declaración no importa, pero revisa el nombre exacto |
